Push and go; easy hosting in gh-pages

27 Mar 2014

Preamble

Four days in and I'm really excited by the github pages / jekyll static site generator combination. I'm not so sure about the all-in-one solution of uploading the unparsed jekyll site because, being a programmer, I have a natural aversion to committing something I'm not happy with and it requires a commit to see how new content gets rendered. But working locally with jekyll running in the background is by far the fastest way to develop a site that I've seen to date. Databases rule for transactional integrity, but it is starting to become clear that the price one pays to get it is really quite high: for tiny data volumes go flat files and a parser, for huge data volumes go hadoop, databases occupy the middle ground.

I think that next time I work on a web project I may well use jekyll to prototype the layout and javascript because it gives such immediate feedback. The teplate syntax is fairly similar to ASP.NET, so once you've got the basic shell working with jekyll then translating that back to ASP is going to be straightforward (this theme originally belonged to the beginnings of an MVC Razor blog).

Developing with github pages and jekyll

Something I realised two days ago is that when hosting in github pages, using jekyll locally to develop the site it is critical they the site kernel (i.e. the thing that jekyll actually builds into your site)

Something to be aware of is that you need to create a new branch to hold the definition as if you mess this up you have no way of going back from the built site to the definition.

One of the most important things programmers do is automate repetitive tasks: we do this because repetition is boring and time is precious. So here is a script that commits the site definition and that built site back to github.

#!/bin/bash

pushd ./

cd [path to site definition]
jekyll build

cd [path to gh-pages branch root]
shopt -s extglob
rm !(.git|.gitignore) -r
cp [path to site definition]/_site/* ./ -r

git add -A .
git commit -a -m 'site updated'
git push

cd [path to site definition]
rm _site -r

git add -A .
git commit -a -m 'site updated'
git push

popd
blog
infrastructure
git

About

This site documents how the bijectiv object-to-object .NET mapping library got/is getting built.

Bijectiv is (being) written by Brian Tyler in his (very limited, see small child to the right) spare time; by day I'm a technical architect developer at Aon's global headquarters in London, the rest of the time I do family stuff.

The purpose of the library is to provide an industrial strength, open source mapper for .NET and the reason for the blog is to describe why it is built the way it is and to act as a resource for other developers who are interested in how a library could be designed and coded.

The blog is written using Jekyll on Mint Linux (apparently that's the way the cool kids do it these days). The library is written in Visual Studio 2013 Ultimate on Windows 8.1 and runs portably in the .NET 4.0 runtime.
